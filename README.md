# Radiology Skills

`radiology-skills` 是一个面向影像组学、影像深度学习和医学影像 AI 研究的 Codex skill。

它的定位是：帮助医生和研究者把真实影像数据、临床问题、模型方法、统计验证、投稿写作和基金申报整理成更规范、更经得起审稿的标准化流程。

适用范围包括 CT、MRI、PET/CT、超声、钼靶、病理影像、多模态影像、传统影像组学、深度学习、基础模型、影像基因组学和影像多组学等。

## 核心定位

这个 skill 主要做四件事：

- 帮你判断一批影像数据适合做什么研究。
- 帮你把影像组学或影像深度学习研究设计得更规范。
- 帮你在写论文、选刊、预审、回复审稿人时发现方法学短板。
- 帮你把影像 AI 研究和基金申报、机制解释、临床转化连接起来。

它更适合做科研设计和质量控制，不适合替代真实建模、正式统计分析、伦理审查或临床决策。

## 证据基础

升级版加入了 2023-2026 年高水平医学影像 AI 文献证据层。

证据层参考了 Radiology、Radiology: Artificial Intelligence、The Lancet Digital Health、Nature Medicine、Nature Communications、eClinicalMedicine、Cell Reports Medicine、The Lancet Oncology、npj Precision Oncology 等期刊中的代表性研究。

它不是完整系统综述，而是一个 PubMed 验证过的种子文献和研究规律库。

总结出的主要规律包括：

- 高水平文章越来越重视真实临床场景，而不只是模型 AUC。
- 外部验证、多中心验证、读者研究和前瞻性评估的重要性明显提高。
- 影像组学文章必须解释 ROI、特征提取、特征筛选、校准和 DCA。
- 影像深度学习文章必须避免患者级泄漏、测试集调参和弱标签训练。
- 影像基因组学必须区分配对数据、非配对公共数据、相关性证据和机制证据。
- foundation model 更适合作为特征提取、迁移学习或基线比较，不适合在普通小样本队列中硬包装成从零训练。

当用户要求“最新文献”“近三年热点”“目标期刊近期发表情况”时，仍应重新联网核验，并写明检索日期。

## 模块说明

### 入口

用于用户只有一个粗略想法时。

它会先整理疾病、模态、样本量、中心数、标签、终点、随访、ROI/mask、训练验证划分、外部验证和临床变量。

输出是一张研究卡，说明这个项目目前能问什么问题、缺什么信息、下一步该进入哪个模块。

### 前沿

用于寻找近三年影像组学和影像深度学习的前沿方向。

它会结合用户的数据条件，判断更适合做疗效预测、预后分层、早筛诊断、分子分型、纵向影像、delta-radiomics、foundation model 迁移学习、读者研究还是临床工作流研究。

它也会明确指出不建议硬做的方向，例如单中心小样本包装成基础模型，或没有分子标签却强行做影像基因组学。

### 证据

用于把建议和近三年高水平文献规律连接起来。

它会总结代表性文章的疾病场景、模态、任务、模型、验证方式、临床评估方式和期刊偏好。

输出不是堆文献列表，而是告诉用户这些文献对当前课题有什么启发、哪些做法值得借鉴、哪些短板会影响投稿。

### 文献

用于系统梳理某个疾病、某个模态或某类任务的文献。

它可以设计 PubMed 检索式，按疾病、影像模态、模型类型、终点、验证方式和期刊层级分类。

输出包括文献框架、研究空白、近年趋势、可引用方向和需要进一步核验的关键文章。

### 组学

用于传统 handcrafted radiomics 研究。

重点检查 ROI/mask 来源、重采样、灰度离散化、滤波、特征提取工具、特征筛选、模型训练、校准、DCA 和外部验证。

它会特别防止标准化、特征筛选、ComBat、阈值选择等步骤发生训练集和测试集泄漏。

### 深度

用于 CNN、Transformer、2D/2.5D/3D 模型、分割模型、检测模型、基础模型和多模态深度学习。

它会帮助设计数据输入、网络路线、迁移学习、数据增强、类别不平衡处理、baseline、ablation、外部验证和失败案例分析。

它会重点审查切片级随机划分、同一患者多序列泄漏、测试集调参、弱标签噪声和可解释性不足。

### 机制

用于影像组学模型或深度学习模型背后的生物学机制解释。

可结合 bulk RNA-seq、单细胞 RNA-seq、空间转录组、基因组学、蛋白组、代谢组、病理组学和免疫微环境。

推荐证据链是：影像表型或 radiomics score -> 分子通路 -> 细胞类型或空间生态位 -> 病理过程 -> 临床终点。

它会严格区分相关性、机制解释、验证证据和因果结论。

### 设计

用于把现有数据转化为完整研究方案。

它会明确临床问题、目标人群、预测任务、标签来源、分析单位、最小可行方案、增强方案、验证路线、主要指标和最大风险。

它的原则是：方法新颖性不能弥补弱标签、弱终点和弱验证。

### 基金

用于国自然、省自然、院内课题和影像 AI 相关基金申报。

它会把论文式表达改成基金式逻辑：临床需求、科学问题、研究假设、研究内容、技术路线、创新点和可行性。

它只润色和重组用户提供的真实基础，不虚构预实验结果、团队条件、合作单位或已获资助。

### 标注

用于 ROI、VOI、mask、病灶选择和读片者流程。

它会审查 2D/3D 标注、whole tumor、largest slice、peritumoral ring、habitat、多病灶策略、读片者年资、盲法、共识和第三方裁决。

常见输出包括标注 SOP、ICC/Dice/Hausdorff 设计、mask 质量控制和 Methods 写法。

### 统计

用于检查样本量、事件数、模型复杂度、变量筛选和评价指标。

覆盖 AUC、PR-AUC、C-index、time-dependent AUC、校准曲线、Brier score、DCA、KM 曲线、Cox 模型和置信区间。

它会判断统计流程是否支持文章结论，尤其关注小样本、低事件数和过度建模。

### 验证

用于判断模型结果是否可信。

它会检查患者级划分、训练/验证/测试集独立性、外部验证、时间验证、阈值选择、超参数调优和失败案例。

高风险问题包括同一患者切片跨集合、病灶跨集合、多期影像跨集合、测试集参与特征选择或阈值选择。

### 多中心

用于多医院、多扫描仪、多协议和多时间段数据。

它会区分 pooled training、center-held-out validation、temporal validation、geographic validation 和真正外部验证。

它会建议 scanner/protocol subgroup、ComBat 或 harmonization 敏感性分析、domain shift 检查和中心间性能报告。

### 公库

用于 TCIA、TCGA、GEO、CPTAC、IDC 等公共数据库。

它会判断公共数据适合做外部验证、配对影像基因组学验证、机制注释、背景支持还是方法对照。

它会特别区分配对数据和非配对数据，避免把非配对公共组学数据写成直接机制验证。

### 规范

用于按高水平期刊常用报告规范审查稿件。

支持 CLAIM、CLEAR、RQS、IBSI、TRIPOD+AI、PROBAST+AI、STARD-AI、CONSORT-AI 和 SPIRIT-AI 等框架。

输出通常是缺项清单、Methods 补充建议、Supplementary Table 建议和投稿前整改优先级。

### 数据

用于整理 DICOM、NIfTI、RTSTRUCT、DICOM-SEG、mask、临床表格和组学数据。

它会检查去标识化、数据来源、数据格式、数据字典、访问限制、共享边界和 Data Availability。

对敏感影像和临床数据，它会优先建议受控访问和合理说明，而不是简单公开。

### 伦理

用于伦理审批、知情同意、隐私保护和数据使用限制。

它会帮助写回顾性研究、知情同意豁免、多中心数据共享、公共数据库使用和隐私保护相关表述。

它不能替代伦理委员会意见，也不会虚构 IRB 编号或审批日期。

### 复现

用于整理研究可复现性材料。

它会检查代码、随机种子、软件版本、模型参数、训练配置、特征表、数据字典、Supplementary Tables 和代码可用性声明。

目标是让审稿人能看清模型是怎么训练、怎么验证、哪些材料能公开、哪些材料受限。

### 写作

用于论文 Methods、Results、Discussion、Abstract、Title 和中文作者笔记转英文投稿表达。

它会优先补足事实，再进行英文润色，避免用漂亮语言掩盖方法学缺口。

如果用户没有提供结果数值、统计方法或伦理信息，它会先列出缺失项，而不是编造内容。

### 图表

用于规划论文主图、补充图、表格和图例。

常见图表包括患者流程图、radiomics pipeline、deep learning pipeline、ROC、PR curve、calibration、DCA、KM、forest plot、SHAP、Grad-CAM 和机制图。

它会让图表服务于研究结论，而不是为了好看而堆图。

### 预审

用于投稿前模拟审稿人和方法学编辑检查。

它会优先找拒稿级问题，例如无外部验证、数据泄漏、标签不可靠、结论过强、统计不足、伦理和数据共享缺失。

输出包括高风险问题、必须修改的问题、可选增强项和建议降低语气的结论。

### 选刊

用于文章基本写完后判断适合投稿的期刊。

它会根据文章类型、创新点、数据规模、验证强度、临床价值、机制深度和近三年期刊偏好给出冲刺、主投、备选和不建议方向。

它不承诺接收，也不会只按影响因子推荐。

### 转化

用于把模型从论文性能推向临床价值评估。

它会设计 reader study、医生+AI 增益、前瞻性验证、真实世界验证、PACS/RIS 工作流、阈值对应临床动作和失败场景分析。

它会区分“模型表现好”和“临床可用”之间的证据差距。

### 回复

用于回复审稿意见和制定修回策略。

常见场景包括审稿人质疑外部验证、样本量、分割流程、统计方法、校准、DCA、数据共享和机制解释。

输出是逐点回复、修改位置、补充分析建议和无法补做时的诚实表述。

### 中文

用于中文作者笔记、中文需求、中文草稿和中英术语转换。

它可以把中文科研想法整理成英文论文结构，也可以把英文审稿意见转成中文行动清单。

它会保留医学术语准确性，避免把中文表达直接机械翻译成英文。

### 依据

用于追溯建议来源。

它会说明建议来自报告规范、方法学原则、近三年文献规律、目标期刊要求还是用户提供的数据事实。

当依据可能更新时，它会提醒重新检索和核验。

## 安装

Windows PowerShell：

```powershell
git clone https://github.com/huang-sir1/radiology-skills.git
cd radiology-skills
New-Item -ItemType Directory -Force "$env:USERPROFILE\.codex\skills" | Out-Null
Copy-Item -Recurse -Force .\radiology-skills "$env:USERPROFILE\.codex\skills\"
```

macOS / Linux：

```bash
git clone https://github.com/huang-sir1/radiology-skills.git
cd radiology-skills
mkdir -p ~/.codex/skills
cp -R radiology-skills ~/.codex/skills/
```

安装后重启 Codex，或重新加载 skills。

## 使用方式

可以显式调用：

```text
Use $radiology-skills to design, audit, or write a radiomics or medical imaging deep learning study.
```

也可以直接中文提问：

- 我有 300 例多中心肝癌 MRI，想找近三年前沿方向并设计课题。
- 根据近三年高水平期刊文献规律，判断这批数据适合做什么影像 AI 课题。
- 影像组学模型做完了，我有 bulk RNA、单细胞和空间转录组，帮我设计机制解析路线。
- ROI 是两位医生勾画的，帮我设计标注 SOP 和 Methods 写法。
- 帮我检查 LASSO、Cox、C-index、校准曲线和 DCA 的统计流程是否规范。
- 投稿前帮我模拟审稿人做一次严格预审。
- 我的文章是单中心回顾性影像组学，没有外部验证，判断适合投什么期刊。
- 我想申报国自然青年基金，帮我润色立项依据、科学问题和技术路线。
- 审稿人说没有外部验证，帮我写回复和修改策略。

## 使用边界

这个 skill 不能替代：

- 真实数据分析和建模。
- 正式统计师审查。
- 医院伦理委员会意见。
- 完整系统综述。
- 临床诊断或治疗判断。
- 目标期刊当天政策核验。

它最适合做的是：研究设计、方法学审查、写作辅助、投稿前自查和科研思路整理。

如果涉及最新文献、最新指南、期刊当前要求或投稿策略，应重新联网检索，并在输出中写明检索日期。
